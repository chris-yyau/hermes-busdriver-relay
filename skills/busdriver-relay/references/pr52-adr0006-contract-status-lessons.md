## Durable workflow lessons

1. **ADR follow-up slices can be machine-readable without becoming mutating.**
   - It is safe to enrich read-only status/capability helpers with ADR design evidence when every authority/capability/dispatch/mutation/finalization/marker-write flag remains false and tests recursively assert no positive authority.
   - Preserve compatibility fields like `contract_adr` when adding richer arrays such as `contract_adrs` / `related_design_adrs`; downstream helpers may already depend on the legacy singleton.

2. **When `hermes-busdriver-deliver --verifier` needs globs, avoid shell-glob assumptions.**
   - Deliver verifier argv is not a shell expansion surface. A verifier like `python3 -m py_compile scripts/hermes-busdriver-*` can fail because `*` is passed literally.
   - Use a deterministic wrapper script (for example a short Python file that expands `Path('scripts').glob('hermes-busdriver-*')` and calls `py_compile.compile`) or an explicit argv list.

3. **Agent-draft postflight ignored-cache blockers can be resolved surgically.**
   - If Codex or a helper starts a repo-local `.codegraph` daemon/cache during the draft, postflight may block on `.codegraph/codegraph.db*`, `daemon.pid`, or `daemon.log` even when tracked diffs and tests are good.
   - Verify the daemon command line is for the current relay repo before terminating it; then remove only the repo-local generated `.codegraph` cache/daemon files. Do not broadly kill all codegraph/Codex processes and do not delete unrelated persistent state.
   - Clean Python `__pycache__` / `.pytest_cache` generated by verification and rerun with `PYTHONDONTWRITEBYTECODE=1` and `pytest -p no:cacheprovider` when postflight needs ignored-file stability.

4. **Reviewer-bot quota comments need interpretation.**
   - CodeRabbit may report a passing status context while leaving a quota/rate-limit PR comment. Treat it as reviewer-state evidence to inspect, but if other current-head reviewers (for example Devin/Cubic) complete with no issues and PR-grind classifies latest head clean, do not invent code findings from the quota notice.

- Focused tests for contract-status and finalization-readiness embedding.
- Static added-line scan and independent read-only Codex review.
- `hermes-busdriver-deliver --mode execute --operation verify` with a deterministic py_compile wrapper.
