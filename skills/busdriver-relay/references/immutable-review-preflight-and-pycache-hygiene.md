# Immutable review preflight and source-tree hygiene

Use this reference for freeze/review loops that invoke external model CLIs against an authenticated immutable boundary.

## 1. Keep Python artifacts outside the candidate tree

Every Python test, helper, and static-compile command must set a private runtime root outside the repo:

```bash
R=/Volumes/Work/.hermes-runtime/<run>
mkdir -p "$R/home" "$R/tmp" "$R/pycache" "$R/pytest"
chmod 700 "$R" "$R/home" "$R/tmp" "$R/pycache" "$R/pytest"
HOME="$R/home" \
TMPDIR="$R/tmp" \
PYTHONPYCACHEPREFIX="$R/pycache" \
PYTHONDONTWRITEBYTECODE=1 \
pytest --basetemp "$R/pytest" ...
```

Apply the same `PYTHONPYCACHEPREFIX` to `python -m py_compile` and standalone Python helpers. `PYTHONDONTWRITEBYTECODE=1` is not a sufficient containment rule for an explicit `py_compile` invocation.

Before freezing, run hygiene and confirm no source-tree `__pycache__`, `*.pyc`, `.pytest_cache`, staged paths, or non-`H` index entries. If a gate detects generated bytecode, remove only the proven generated cache directories, rerun that failed gate once, and preserve the retry evidence rather than rewriting the original result.

## 2. Preflight real inference before formal lanes

Before creating formal `START` closure artifacts or spending a review lane:

1. Use the exact reviewer binary, model, endpoint, sandbox flags, and credential topology planned for the formal runner.
2. Execute a minimal no-tools/no-write inference smoke such as “return exactly REVIEWER_SMOKE_OK.”
3. Treat a CLI `auth status` command as diagnostic only; it does not prove that token refresh, endpoint policy, quota, or model inference works.
4. Start the immutable formal lane only after the smoke returns valid model output.

This avoids confusing installation/auth metadata with actual review availability.

## 3. Isolate mutable state without hiding the configured credential source

Keep `TMPDIR`, `XDG_CACHE_HOME`, `XDG_CONFIG_HOME`, `XDG_DATA_HOME`, logs, and model scratch under the private runtime root. When a provider intentionally stores OAuth in the native HOME or OS keyring, retain the native credential lookup path rather than copying secrets into a fresh HOME. Never print, copy, or persist token values.

The formal review remains read-only because its review view is authenticated and immutable; credential lookup and mutable cache containment are separate concerns.

## 4. Preserve failed attempts and frozen identity

- Never overwrite a failed reviewer artifact. A retry uses a new lane or suffixed report/log/result files while pointing at the same authenticated immutable view.
- Retry only after correcting the preflight condition. If the repaired retry succeeds, stop retrying.
- If inference remains unavailable, mark the lane blocked rather than synthesizing a verdict or substituting a different reviewer identity silently.
- Run an `END` closure after a blocked attempt to prove that the live source digest, candidate tree identity, dirty counts, and index state did not drift.
- Do not commit, push, open a PR, or merge until every required reviewer has a valid verdict and start/end closure on the same immutable identity.

## 5. Evidence discipline

Record separately:

- reviewer identity and invocation route;
- exact immutable source digest and candidate tree;
- inference exit status and report existence;
- failed-attempt logs;
- retry/adjudication output;
- final `END` closure.

A model-independent fallback may supply additional evidence, but it does not satisfy a lane whose policy requires a named provider/model unless the workflow explicitly authorizes that substitution.
