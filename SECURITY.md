# Security Policy

## Reporting a Vulnerability

Please report suspected security vulnerabilities privately to
**security@diveanddev.com**. Do not open a public issue for security reports.

Include where possible: affected script/workflow, reproduction steps, and impact.

## Response Timeline

- **Acknowledgment:** within 48 hours
- **Initial assessment:** within 1 week
- **Fix / mitigation:** prioritized by severity once confirmed

## Scope

All code in this repository, its CI/CD workflows, dependencies, and the
Hermes-owned runtime state it manages. This is a read-only / fail-closed relay:
reports about probes writing outside their declared boundaries are in scope.

## Recognition

We credit reporters who follow responsible disclosure in release notes, unless
anonymity is requested.
